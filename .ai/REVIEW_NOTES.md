# Pipeline Review Notes

## Fixed

- CRITICAL | .ai/pipeline/tools/filesystem.py | (C1) Path traversal: `except ValueError: pass` allowed writes to any filesystem path when the resolved target fell outside `REPO_ROOT`. The `except` clause now raises `PermissionError` instead, blocking absolute paths like `/etc/crontab` and traversal patterns like `../../../etc/passwd` from bypassing the protected-paths check.
- HIGH | .ai/pipeline/tools/filesystem.py | (H1) Liquibase protection was blocking ALL writes under `apps/api/src/main/resources/db/changelog/`, including new migration files that the BE engineer must create per its persona. Protection now blocks only modification of files that already exist; new changeset files are permitted.
- HIGH | .ai/pipeline/tools/filesystem.py + .ai/pipeline/agents/engineers/fe_agent.py + .ai/pipeline/agents/engineers/be_agent.py | (H2) Case-sensitive `startswith` allowed alternate-casing bypasses on Windows (e.g. `Apps/Web/Src/Components/Ui/…`). All protected-path comparisons now lowercase both sides before matching, including the redundant shadcn/ui check in `fe_agent.write_project_file` and the backend-path cross-check in `be_agent.write_project_file`.
- HIGH | .ai/pipeline/agents/planning/po_agent.py, architect_agent.py; .ai/pipeline/agents/engineers/be_agent.py, fe_agent.py; .ai/pipeline/agents/quality/review_agent.py, secops_agent.py, verification_agent.py | (H3) LLM prompts and raw return values were not captured in `.ai/logs/{task_id}.log`, making post-hoc debugging of malformed agent outputs impossible without rerunning the pipeline. Each agent's `run()` (or `_run_auto_fix_agent` for verification) now emits two DEBUG log lines via the shared task logger: the truncated prompt (800 chars) and the truncated `repr(raw)` of the agent's return value (500 chars). `task_id` is threaded through `_check_with_llm_fix` -> `_run_auto_fix_agent` so the verification fixer can also log to the per-task file.
